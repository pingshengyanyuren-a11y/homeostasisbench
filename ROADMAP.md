# Roadmap

## v0.1

Current focus:

- reproducible benchmark suite with `normal`, `overload`, and `failure_storm`
- baseline vs bio-inspired comparison
- layer ablation and acceptance gates
- package install, CLI entrypoint, and generated reports

## v0.2

Make the comparison harder to dismiss:

- add stronger non-biological baselines
- add parameter sweeps for backlog-heavy and false-signal-heavy regimes
- publish a standard result card for every controller

## v0.3

Turn the benchmark into a contribution surface:

- controller adapter API
- plug-in registry for custom schedulers and policies
- reproducible seed packs and scenario manifests

## v0.4

Move closer to realistic agent workloads:

- task decomposition and inter-agent message costs
- heterogeneous worker capabilities and tool-use profiles
- memory/context fragmentation as a first-class metabolic cost

## v0.5

Make it community-friendly:

- public leaderboard
- benchmark report template
- contribution guide for new controllers and scenarios
- release cadence with tagged benchmark snapshots
